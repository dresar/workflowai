## 1. Executive Summary
Project 7/20/2026 adalah platform **Biolink** yang memungkinkan pengguna (content creator, UMKM, affiliate marketer) membuat halaman profil publik yang terintegrasi dengan kumpulan tautan (link) dan katalog produk dalam tampilan grid 2 kolom yang dioptimalkan untuk perangkat seluler. ### 2.1 Problem Statement

### 6.3 Row Level Security (RLS)
```sql
-- Aktifkan RLS
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE links ENABLE ROW LEVEL SECURITY;
ALTER TABLE products ENABLE ROW LEVEL SECURITY;
ALTER TABLE clicks ENABLE ROW LEVEL SECURITY;

-- Policy untuk public read (halaman biolink)
CREATE POLICY "Public profiles are viewable by everyone." ON profiles
    FOR SELECT USING (true);

CREATE POLICY "Public links are viewable by everyone." ON links
    FOR SELECT USING (is_active = true);

CREATE POLICY "Public products are viewable by everyone." ON products
    FOR SELECT USING (is_active = true);

-- Policy untuk owner write (admin panel)
-- Catatan: auth.uid() disinkronkan dengan id profil di Supabase Auth dengan menggunakan fungsi custom claim atau mapping.
-- Asumsikan ada tabel user_profiles yang menghubungkan auth.uid() ke profiles.id atau gunakan RLS function.
-- Alternatif sederhana: gunakan service role di server actions untuk menghindari RLS pada operasi tulis. Di sini kita demonstrasikan dengan asumsi uid = profile_id.
CREATE POLICY "Users can update own profile." ON profiles
    FOR UPDATE USING (auth.uid() = id);

CREATE POLICY "Users can insert their own links." ON links
    FOR INSERT WITH CHECK (auth.uid() = profile_id);
CREATE POLICY "Users can update own links." ON links
    FOR UPDATE USING (auth.uid() = profile_id);
CREATE POLICY "Users can delete own links." ON links
    FOR DELETE USING (auth.uid() = profile_id);

-- Produk serupa
CREATE POLICY "Users can manage own products." ON products
    FOR ALL USING (auth.uid() = profile_id);
```

**GET /api/v1/public/profiles/{username}**
- Headers: `x-api-key: <api_key>`
- Middleware: Validasi API key, tarik profile_id terkait.
- Response: Sama seperti endpoint publik, tetapi hanya jika API key milik pemilik profil yang sama? Atau untuk semua profil? Untuk fleksibilitas, kita batasi API key hanya bisa mengakses profil pemiliknya (seperti personal token). Jadi endpoint ini akan mengembalikan data profil si pemilik API key, bukan berdasarkan username. Alternatif: Admin membuat API key yang bisa mengakses profil miliknya sendiri. Desain akhir: API key terikat ke profile_id, dan endpoint `/api/v1/public/profile` (tanpa username) mengembalikan data profil pemilik. Atau jika ingin mengakses profil orang lain, perlu mekanisme OAuth. Untuk v1, kita batasi akses hanya untuk profil sendiri.
- Response 200: Data profil, links, products.
- Response 401: Invalid API key.
